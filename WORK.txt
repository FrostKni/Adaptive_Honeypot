##Problem Statement:

To design a fully autonomous, containerized adaptive honeynet capable of dynamically modifying its environment, escalating interaction based on attacker actions, and preserving active sessions to maximize engagement and intelligence value.



##Scope of Work:
The design, implementation, and evaluation of an autonomous adaptive
honeynet using Docker containers and AI-driven orchestration. The major tasks include:

1. Designing modular honeypot templates (SSH/Cowrie).
2. Building an automated engine for deploying, reconfiguring, and migrating honeypots based on attacker behaviour.
3. Developing a log-processing pipeline with anonymization and artefact extraction.
4. Using LLMs and rule-based reasoning to classify attacker actions and recommend
interaction-level escalation.
5. Creating a dynamic configuration generator for the new docker containers.
6. Implementing session-preserving migration between low- and high-interaction honeypots.
7. Designing a dashboard for orchestration and behaviour visualization.
8. Benchmarking system performance, deception effectiveness, and attacker engagement.
This scope ensures the system is practically deployable and supports defence-oriented cyber operations.


Automated monitoring of container health, resource usage, and
activity enables restarting, replacing, or tuning honeypot components.

Quantitative evaluation: The system is benchmarked for deception effectiveness,
engagement duration, behavioural diversity, and configuration accuracy against static
honeypots.
